Gemini + Cursor

Nubank Training Session

IA prática para o dia a dia de produto

Duração: 60–70 min • Facilitadora: Rayssa

Agenda

  • Introdução (10m): objetivos, papéis e expectativas
  • Gemini (20m): recursos práticos para produto (sem código)
  • Cursor (30m): como acompanhar e colaborar com engenharia
  • Encerramento (10m): recapitulação, próximos passos e Q&A
Contextualizar: por que agora, benefício x risco, governança. Setar tom interativo.

Objetivos

  • Entender quando usar Gemini vs Cursor
  • Usar o Gemini para resumir, planejar e priorizar sem código
  • Ver como o Cursor ajuda a transformar ideias em entregas e acompanhar mudanças
  • Conhecer o passo a passo para solicitar acesso ao Cursor

Gemini — além do básico

Recursos-chave

  • Saída estruturada (tabelas/planilhas)
  • Formatos e regras claras (prioridade, prazos, responsáveis)
  • Usar arquivos como base (PRD, pesquisas, planilhas)
  • Multimodal: imagens de fluxo, PDFs
  • Auto-checagem de critérios de sucesso

Prompt patterns

  1. Contexto → Objetivo → Restrições
  2. Formato de saída (JSON, tabela)
  3. Critérios de qualidade/aceitação
  4. Passo final: valide requisitos
Mostrar 1–2 exemplos rápidos. Reforçar política de dados segura.

Roteiro de demo (Gemini)

  1. Gerar 3 opções e uma matriz de decisão em tabela
  2. Analisar um fluxo (imagem) e listar melhorias em linguagem simples
  3. Transformar PRD em plano de execução (marcos, riscos, métricas)
  4. Extra: resumir reunião e lista de ações a partir de transcript
Exemplo de prompt
Contexto: [problema do time]
Objetivo: 3 opções; prós/contras; recomendação
Formato: tabela com colunas {opção, benefícios, riscos, esforço, pontuação}
Valide: cubra objetivos e aponte perguntas em aberto

Cursor — visão geral

  • Ambiente de trabalho usado pela engenharia com assistente de IA
  • Mostra antes/depois das mudanças para revisão segura
  • Acelera criação de documentação e melhoria de qualidade
  • Conecta-se ao histórico de versões do projeto

Tour rápido (Cursor)

Recursos

  • Conversa sobre o projeto para pedir mudanças
  • Sugestões de melhoria e documentação automática
  • Visualização de antes/depois para aprovar
  • Resumos claros do que mudou

Boas práticas

  • Peça mudanças pequenas e objetivas
  • Defina critérios de aceitação em linguagem de negócio
  • Nunca compartilhe dados sensíveis
  • Confira o resultado antes de aprovar

Exemplo prático observado (Cursor)

  1. Transformar um PRD em documentação do projeto padronizada
  2. Gerar checklist de lançamento com responsáveis e prazos
  3. Criar/atualizar a FAQ/README do projeto
  4. Sugerir melhorias de fluxo a partir de uma captura de tela
  5. Preparar mensagens de atualização para stakeholders
Conduzir ao vivo; equipe sem acesso observa e anota.

Como o Cursor complementa o Gemini

Gemini

  • Ideação, análise, pesquisa
  • Multimodal e saídas estruturadas
  • Validação de critérios de sucesso

Cursor

  • Transformar ideias em mudanças reais no produto
  • Antes/depois auditável e fácil de revisar
  • Documentação viva durante a execução

Solicitar acesso ao Cursor

  1. Abra ticket para TI/Security/Legal (uso, repos, dados)
  2. Instale no macOS (site ou Homebrew, se permitido)
  3. Configuração segura: providers, chaves corporativas, Git
  4. Validação em sandbox: experimento pequeno e mensurável
brew install --cask cursor  # se sua política permitir
# Configurações: desabilite telemetria não essencial; use chaves corporativas

Boas práticas de prompt

  • Contexto → Objetivo → Restrições → Formato → Critérios
  • Peça validação como último passo
  • Para entregas: passos pequenos + critérios + mensagem de resumo

Riscos e mitigação

  • Dados: anonimização e ambientes isolados
  • Chaves/API: corporativas e rotacionáveis
  • Qualidade: revisão humana, testes, linters
  • Compliance: retenção off quando possível, logs de auditoria

Encerramento & Próximos passos

  • Ticket de acesso ao Cursor + materiais
